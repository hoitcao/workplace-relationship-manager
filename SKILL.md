# Workplace Relationship Manager

帮助曹总高效管理职场人际关系，覆盖：智能消息回复撰写、跨团队协作推动、拒绝甩锅技巧、待办追踪提醒。

## 何时使用此 Skill

- 需要给某个协作者回复消息，想让语气和策略更有针对性
- 需要推动某人（研发/设计/市场等）完成任务但不知道怎么开口
- 需要拒绝别人甩来的锅，同时维护关系
- 需要在沟通中记录承诺/待办，并后续跟进
- 需要维护某个人的基本档案，以备后续对话参考

---

## 工作模式

本 Skill 有三种工作模式，每次激活时先判断用户意图：

| 模式 | 触发场景 | 输出 |
|------|----------|------|
| **撰写回复** | 用户粘贴对方消息，希望 AI 帮助起草回复 | 回复草稿 + 策略说明 + 待办提取 |
| **档案管理** | 用户新增/更新某人的背景信息 | 更新/创建联系人档案 |
| **待办查看** | 用户询问某人/某事的跟进状态 | 待办清单 + 最近互动摘要 |

## 模式一：撰写回复

### 输入要素

执行回复撰写前，收集以下信息（可分步追问，不要一次列出所有问题）：

**必需信息：**
1. **对方是谁** —— 姓名/昵称（用于调取档案，若无档案则当场收集基本信息）
2. **对方发的消息** —— 原文粘贴

**可选但有价值：**
3. **当前背景** —— 这条消息发生在什么项目/事件背景下
4. **你的意图** —— 你希望这条回复达到什么效果（推动对方行动 / 拒绝请求 / 澄清误解 / 普通协作）

### 处理流程

```
1. 读取联系人档案（contacts/<姓名>.md）
   └─ 若档案不存在 → 询问必要信息后先创建档案

2. 【必须执行】读取对话历史
   └─ 阅读档案中"对话线索"区块的所有条目
   └─ 理解：我们之前谈过什么？达成过什么？对方承诺了什么？上次状态是什么？
   └─ 如有未解决的待办或悬而未决的事项，在出建议前先提示曹总

3. 结合对话历史 + 当前消息，分析对方的真实意图和潜台词
   └─ 例如：对方说"这周可能来不及"→ 结合历史，这是第几次说来不及？之前怎么处理的？

4. 结合档案中的"性格特点"和"沟通偏好"选择策略
5. 调用策略库（references/reply-strategies.md）匹配对应场景

6. 撰写回复草稿

7. 【必须执行】压缩检查：
   - 能不能再短？删掉铺垫、客套、废话
   - 第一句话是否说清了目的/请求？
   - 有没有哪句去掉后意思没变？有则删

8. 提取本次对话中涉及的待办事项 → 追加到档案待办区

9. 【必须执行】将本次对话写回档案的对话线索区块
   └─ 格式见"对话线索写回规范"（下方）

10. 输出：回复草稿 + 策略说明 + 本次提取的待办
```

### 对话线索写回规范

每次完成回复建议后，将本次对话追加到档案的 `## 对话线索` 区块：

```markdown
### 线索 #N：[一句话概括这件事的主题]
| 时间 | 方向 | 内容摘要 | 状态 |
|------|------|----------|------|
| YYYY-MM-DD | 对方→我 | [对方说了什么，核心意图] | |
| YYYY-MM-DD | 我→对方 | [我回复了什么，诉求是什么] | 等待回复 🟡 |
```

**状态说明：**
- `等待回复 🟡`：我已发出，等对方回
- `对方已回 ✅`：对方有了明确答复
- `已推进 ✅`：事情已有进展或解决
- `搁置 ⏸️`：暂时不推进
- `超时未回 🔴`：超过约定时间未收到回复，需跟进

**新消息到来时的更新方式：**
在对应线索下追加一行，不新建线索；
只有当完全是一件新的事时，才新建线索。

### 输出格式

```markdown
## 📝 回复草稿

[回复正文，可直接复制使用]

---

## 💡 策略说明
- 语气选择：[为什么用这个语气]
- 关键策略：[例如：先认可再推进 / 转移责任锚点 / 设定明确截止时间]

---

## 📌 本次提取的待办
- [ ] [具体待办] —— 负责方：[你/对方] —— 截止：[时间，若有]
```

## 模式二：档案管理

### 联系人档案结构

每位联系人对应一个文件：`contacts/<姓名或昵称>.md`

档案模板详见：`references/contact-profile-template.md`

**核心字段：**

```yaml
姓名: 
角色: [领导 / 研发 / 设计 / 市场 / 公关 / 运营 / 其他]
部门/团队: 
与我的关系: [上级 / 平级协作 / 下游 / 外部合作方]

性格特点:
  - 
沟通偏好:
  - [例如：偏好数据说话 / 情绪敏感型 / 雷厉风行 / 官僚化需要走流程]
  
工作习惯:
  - [例如：回消息慢 / 喜欢开会对齐 / 更倾向文字沟通]

推动技巧:
  - [例如：给他设定明确截止时间会有效 / 需要先让领导背书再推]

拒绝甩锅技巧:
  - [例如：直接拒绝会翻脸，需要给台阶 / 可以直接说清边界]

注意事项:
  - [例如：在领导面前很维护面子 / 跨部门合作中有话语权]
```

### 创建/更新档案

当用户描述一个新联系人时：
1. 主动追问关键信息（分批，不要一次问太多）
2. 在 `contacts/` 目录下创建对应 Markdown 文件
3. 确认保存后告知用户，后续回复该联系人时会自动调取

## 模式三：待办追踪

### 待办的存放位置

待办统一追加在 `contacts/<姓名>.md` 文件末尾的 `## 待办事项` 区块。

### 待办格式

```markdown
## 待办事项

### 进行中
- [ ] [具体事项] | 负责方：我 | 截止：YYYY-MM-DD | 来源：[对话日期简述]
- [ ] [具体事项] | 负责方：对方 | 截止：YYYY-MM-DD | 来源：[对话日期简述]

### 已完成
- [x] [具体事项] | 完成于：YYYY-MM-DD
```

### 查看待办

用户可问：
- "XXX 那边有什么待办？" → 读取对应档案的待办区
- "最近有哪些跨部门的跟进事项？" → 扫描所有 contacts/*.md 提取未完成待办汇总

## 核心策略框架

本 Skill 内置以下场景策略，详见 `references/reply-strategies.md`：

### 场景 A：推动对方行动（催活）
- 不激进，给具体时间节点
- 附上"不推进会有什么影响"的背景，让对方感受到必要性
- 必要时抄送领导或请领导背书（视关系而定）

### 场景 B：拒绝甩锅 / 划清边界
- 先认可对方的压力和困难（减少对抗情绪）
- 清晰说明"这件事不在我的职责范围内的原因"
- 可以给一个"有限协助"的折中方案，保留善意

### 场景 C：向上沟通（与领导）
- 带结论说话，不要只抛问题
- 提供选项，而不是让领导做所有判断
- 有数据/事实支撑时更有说服力

### 场景 D：平级协作对齐
- 明确各自的责任边界
- 设定里程碑和检查点，不要只有终点
- 书面确认，避免"他说我说"的扯皮

### 场景 E：敏感/情绪化沟通
- 先处理情绪，再处理事情
- 用 SBI 模型（Situation-Behavior-Impact）组织语言
- 给对方留台阶，不当众让人难堪

## 与其他 Skill 的协作

- **`difficult-workplace-conversations`**：遇到复杂职场冲突场景时，调用该 Skill 的"准备-交付-跟进"框架做深度准备
- **`feedback-mastery`**：需要给下属或协作方正式反馈时，调用 SBI 模型生成结构化反馈

## 使用示例

### 示例 1：推动研发上线

```
用户：张工发消息说"这周可能来不及"，帮我回一下
AI：（调取张工档案）→ 分析话语 → 起草"定时间节点+说明影响"的推进回复 → 记录待办
```

### 示例 2：拒绝市场甩锅

```
用户：市场小李说"你们产品这边能不能帮我们出一版文案"，我不想接这活
AI：（调取小李档案）→ 分析诉求 → 起草"先认可+明确边界+给折中方案"的回复
```

### 示例 3：新建档案

```
用户：帮我记录一下我们 PR 的王姐，她比较强势，不喜欢被催
AI：追问补充信息 → 创建 contacts/王姐.md → 后续对话自动调取
```

### 示例 4：查看待办

```
用户：设计那边有什么待跟进的？
AI：扫描 contacts/ 中设计相关联系人 → 汇总未完成待办
```
