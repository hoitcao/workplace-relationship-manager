# 待办追踪规范

---

## 待办来源识别

每次处理回复后，从对话中提取：

| 类型 | 说明 |
|------|------|
| 我的承诺 | 回复中我说会做的事 |
| 对方承诺 | 对方说会完成的事 |
| 待确认 | 发出后等对方回复/拍板 |
| 跟进提醒 | X天后无响应需再催 |

---

## 待办格式

```
- [ ] 事项描述 | 负责：我/对方 | 截止：YYYY-MM-DD | 线索：#N
- [x] 已完成事项 | 完成于：YYYY-MM-DD
```

---

## 全局汇总格式

扫描所有 `contacts/*.md` 后，按以下结构输出：

```
📋 待办汇总（YYYY-MM-DD）

🔴 紧急（3天内）
- [ ] 事项 — [姓名] | 截止：日期

🟡 本周内
- [ ] 事项 — [姓名] | 截止：日期

🟢 本周后
- [ ] 事项 — [姓名] | 截止：日期

⏳ 等待对方回复
- [ ] 事项 — 等待：[姓名] | 发出于：日期
```

---

## 完成标记

用户说"XXX 那个事情完成了" → 在对应档案将该条改为 `[x]` 并补完成日期，移至已完成区块。
