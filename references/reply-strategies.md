# 回复策略库

按需加载。包含五大场景的策略原则和话术模板。

---

## 🔴 最高优先级：精炼简洁准确

所有草稿生成后必须执行压缩自检：

1. 只说必须说的，删掉铺垫和重复
2. 结论前置，第一句说清目的或请求
3. 用词精准——"4月10日前"而非"尽快"；"确认方案A/B"而非"你看一下"
4. 句子超30字考虑拆短或用列表
5. 礼貌词适度保留（每条1-2处），不堆砌，不冷漠

## 场景 A：推动对方行动（催活）

**核心原则：** 不要只催"快点"——给截止时间 + 说清影响，让对方感受到必要性而非你的个人焦虑。给一个简单的响应出口（"只需回复时间节点即可"）。

**轻推版**（第一次，关系较好）
```
XXX，[事项]现在进展到哪了？我这边[时间]前需要，方便给个大概时间节点吗？
```

**中推版**（等待较久，平级）
```
XXX，[事项]这边跟进一下，我们[节点]计划[时间]前推进，你那边大概什么时候能给到？有卡点我们一起看。
```

**强推版**（紧急或多次拖延）
```
XXX，这件事比较急，[截止时间]前需要你确认[具体内容]，否则会影响[后续环节]。麻烦今天内给个明确答复。
```

**请领导背书**：私下提醒无响应后，在相关群同步进展，自然形成压力。

## 场景 B：拒绝甩锅 / 划清边界

**核心原则：** 先共情再边界，不要上来就说"这不是我的事"。给理由（不是态度问题），给折中方案（保留善意），涉及责任认定的留书面记录。

**温和边界版**（初次，关系较好）
```
XXX，我理解你这边压力大。不过[具体工作]按分工在[其他团队]那边，我这边没有对应资源/权限。我可以帮你联系[负责人]，你看行吗？
```

**明确划界版**（已有先例或涉及责任）
```
XXX，这块职责在[XX部门]，不在我们这边。建议直接找[负责人]推进更快。我可以配合提供[具体信息]，但主责不在我这里。
```

**拒绝背锅版**（责任认定）
```
XXX，我看了一下，[事项]当时的决策是[事实]，执行是[谁负责]。我这边[说明自己的处理]。这块责任需要准确定位，不然后续改进也没方向，你觉得呢？
```

## 场景 C：向上沟通（与领导）

**核心原则：** 带结论上去，不抛问题。给2-3个选项让领导拍板。有数据更好判断。要资源时说清"不给会怎样"。

**汇报进展**
```
[领导]，[项目]进展更新：[具体进展]，目前[状态]。计划[时间]前完成[下一步]。
```

**请示决策**
```
[领导]，这件事有两个方向：
方案A：[内容]，优点[XX]，风险[XX]
方案B：[内容]，优点[XX]，风险[XX]
我倾向A，因为[核心理由]。您看？
```

**申请资源**
```
[领导]，[事项]需要协调[资源]，原因是[现状和影响]。到位后预计能[效果]。麻烦帮忙协调一下。
```

## 场景 D：平级协作对齐

**核心原则：** 口头说的不算数，重要的书面确认。对齐"完成"的定义（你以为的完成≠他以为的完成）。有中间检查点，不只有终点。

**分工确认**
```
XXX，对一下[事项]分工：
- 我负责：[内容] → [时间]前给到
- 你负责：[内容] → [时间]前

没问题吧？有调整我们现在说。
```

**书面确认共识**
```
XXX，把今天的共识总结一下，你确认：
1. [共识点]
2. [共识点]

没问题我们就按这个推进了。
```

## 场景 E：敏感 / 情绪化沟通

**核心原则（SBI 模型）：** 先处理情绪再处理事情。针对行为不评价性格。给对方留台阶。

**SBI 结构**
```
[情境]的时候，当[具体行为]，对[我/项目/团队]造成了[影响]。我想聊一下，看能不能找到更好的方式。
```

**示例**
```
上周评审会上，当你说"这个设计完全不可用"时，设计同学当时很措手不及，也影响了后续讨论。我觉得可以找个方式让反馈更有建设性，你觉得呢？
```

## 通用原则

1. 对方情绪激动时，先让情绪降温，不要推进实质问题
2. 重要的事聊完发文字确认，不能只口头
3. 催的时候说影响，不说自己的着急
4. 拒绝 = 拒绝 + 理由 + 折中，不能只有"不"
5. 对领导永远带结论，汇报问题同时带建议
6. 每条回复过压缩三问（见文件顶部）
