# 回复策略库

本文档是 workplace-relationship-manager Skill 的核心策略参考，包含各类职场消息场景的应对策略和话术模板。

---

## 🔴 最高优先级原则：精炼、简洁、准确

**所有回复草稿，必须在完成内容后执行一次"压缩检查"，确保符合以下标准：**

1. **只说必须说的** —— 删掉所有铺垫、客套、重复的话；对方能推断出来的不说
2. **一条消息只传达一个核心意图** —— 如果有多件事，拆开发，不要堆在一条里
3. **结论前置** —— 先说结论/请求，再说理由；不要让对方读到一半才知道你要什么
4. **用词精准，不模糊** —— 用"4月10日前"代替"尽快"；用"需要你确认方案A/B"代替"你看一下"
5. **礼貌词适度保留，不堆砌** —— 每条消息保留一两处礼貌词即可（如"麻烦"、"谢谢"），让语气不显得冷硬；但不要开头结尾全是客套，冲淡重点
6. **句子短** —— 一句话超过30字时，考虑拆成两句或用列表

**压缩自检三问（每次出草稿后过一遍）：**
- ❓ 这条消息能不能再短10个字？
- ❓ 对方读完第一句，知不知道我要什么？
- ❓ 有没有哪句话去掉之后意思没变？有的话，删掉。

## 场景 A：推动对方行动（催活）

### 适用情况
- 对方迟迟没有进展或没有回复
- 需要对方在某个截止时间前完成某件事
- 对方的拖延已经影响了你的工作推进

### 策略原则
1. **不要只催"快点"** —— 给出明确截止时间，对方才知道边界
2. **说清楚影响** —— "你不完成→我会怎样"，让对方感受到必要性，而不是你的个人焦虑
3. **给一个简单的响应出口** —— 让对方知道"只需要回复 XX 就算配合了"
4. **必要时请领导或上级背书** —— 某些人只有看到更高权威才会动

### 话术模板

**轻推版（第一次提醒，关系较好）：**
```
XXX，上次说的[具体事项]，现在进展怎么样了？我这边[XX 时间]之前需要，你方便的话给我一个大概时间节点吗？
```

**中推版（已经等待一段时间，关系平级）：**
```
XXX，[具体事项]这块我跟进一下，我们[项目/节点]计划在[时间]前推进，你那边大概什么时候能给到？如果有什么卡点我们可以一起看一下。
```

**强推版（比较紧急或对方多次拖延）：**
```
XXX，这件事比较紧急，[截止时间]之前我需要你确认[具体内容]，否则会影响[后续环节/项目时间线]。麻烦今天内给我一个明确的答复，谢谢。
```

**请领导背书版：**
```
[先私下提醒一次，若无回应则]在[领导/群]里同步进展，自然形成压力。
```

## 场景 B：拒绝甩锅 / 划清边界

### 适用情况
- 对方把本不属于你的工作推过来
- 对方出了问题想让你来背
- 对方要求你协助但超出了合理边界

### 策略原则
1. **先共情，再边界** —— 不要上来就说"这不是我的事"，会激化对抗
2. **解释为什么不在你职责范围内** —— 给理由，让对方看到不是态度问题
3. **给一个有限协助的折中** —— "我不做主，但我可以帮你找到对的人"
4. **书面留记录** —— 涉及责任认定的内容，记得有文字记录

### 话术模板

**温和边界版（关系较好，初次甩锅）：**
```
XXX，我理解你这边压力比较大。不过这块[具体工作]按照分工来说在[其他团队/角色]那边，我这边没有拿这块的资源/权限。我可以帮你牵个线联系[对应负责人]，你看这样行吗？
```

**明确划界版（已有先例或涉及责任）：**
```
XXX，这件事我之前了解过，这块的职责范围是归属[XX部门/角色]的，不在我们这边。我建议直接找[对应负责人]推进，会更快。这边如果需要我配合提供[具体信息]，我可以支持，但主责不在我这里。
```

**拒绝背锅版（问题责任认定）：**
```
XXX，我看了一下这个情况，[具体事项]当时的决策是[还原事实]，执行是[谁负责的]。我这边[说明自己的处理情况]，这块的责任我觉得我们需要准确定位，不然后续改进也没有方向。你觉得呢？
```

## 场景 C：向上沟通（与领导）

### 策略原则
1. **带结论说话** —— 先说结论，再解释过程，不要让领导猜你要说什么
2. **提选项，不抛问题** —— "有 A/B 两个方案，我建议 A，因为..."比"这件事怎么办"更有价值
3. **数据和事实优先** —— 有具体数字和案例，领导更好判断
4. **要资源时说清楚为什么要/不要会怎样** —— 让领导知道你在为他省麻烦

### 话术模板

**汇报进展版：**
```
[领导]，[项目名]这边进展更新一下：[具体进展]，目前[状态]。接下来计划在[时间节点]前完成[下一步]。
```

**请示决策版：**
```
[领导]，这件事有两个方向可以走：
方案 A：[具体内容]，优点是[XX]，风险是[XX]
方案 B：[具体内容]，优点是[XX]，风险是[XX]

我倾向 A，主要是[核心理由]。您这边有什么想法？
```

**申请资源版：**
```
[领导]，[项目/事项]这块我需要协调[具体资源]，主要原因是[现状和影响]。如果能拿到这个支持，预计可以[带来的效果]。可以麻烦您帮忙协调一下吗？
```

## 场景 D：平级协作对齐

### 策略原则
1. **明确各自责任边界** —— 口头说的当不了数，重要的事书面确认
2. **设定里程碑，不只有终点** —— 中间有检查点，出问题可以早发现
3. **对齐"完成"的定义** —— 你以为的完成和他以为的完成可能不一样
4. **及时同步变化** —— 你这边有任何影响对方的变化，提前说

### 话术模板

**任务分工确认版：**
```
XXX，我们来对一下[项目/事项]的分工：
- [你负责]：[具体内容] → 计划在[时间]前给到
- [对方负责]：[具体内容] → 需要在[时间]前

这样理解没问题吧？有什么要调整的我们现在说。
```

**对齐验收标准版：**
```
XXX，[事项]我们先对一下验收标准，我理解"完成"的意思是[具体定义]，你这边也是这样理解的吗？避免到时候返工。
```

**书面确认版（重要决定）：**
```
XXX，根据今天的讨论，我把我们的共识总结一下发给你，你确认一下：
1. [共识点 1]
2. [共识点 2]
3. [共识点 3]

如果没问题，我们按这个推进了。
```

## 场景 E：敏感 / 情绪化沟通

### 策略原则（基于 SBI 模型）
1. **先处理情绪，再处理事情** —— 对方有情绪时，先表达理解再说实质
2. **用 SBI 组织语言** —— Situation 情境 → Behavior 行为 → Impact 影响
3. **对事不对人** —— 针对具体行为，不评价性格
4. **给对方留台阶** —— 别把人逼到墙角，要有退路

### SBI 话术结构
```
"[情境]的时候，当[具体行为]，对[我/项目/团队]造成了[影响]。我想跟你聊一下这个，看看我们能不能找到更好的方式。"
```

### 示例
```
"上周评审会上，当你说'这个设计完全不可用'的时候，设计同学当时有些措手不及，也影响了后续讨论的氛围。我觉得我们可以找个方式，让反馈更有建设性，你觉得呢？"
```

## 根据性格类型调整策略

| 性格类型 | 特征 | 对应策略 |
|----------|------|----------|
| **雷厉风行型** | 说话直接、效率优先、不喜欢废话 | 直接说结论，省掉铺垫，不要绕弯 |
| **情绪敏感型** | 在意被尊重，容易有情绪 | 先共情再说事，给面子，避免群里直接点名 |
| **官僚流程型** | 做事按规矩、喜欢走正式流程 | 尊重流程，提前申请，不要绕过他 |
| **数据驱动型** | 以事实和数据说话 | 带数据、带案例来对话 |
| **社交型** | 重视关系，喜欢先聊聊 | 先建关系再谈事情，定期维护 |
| **谨慎保守型** | 做决定慢，怕担责任 | 帮他分析风险，提供背书或先例，降低决策压力 |

## 通用沟通原则

1. **不要在对方情绪激动时推进实质问题** —— 先让情绪降温
2. **重要的事情不要只口头说** —— 聊完发一条文字确认
3. **催的时候说影响，不说自己的着急** —— "影响项目"比"我很急"有用
4. **拒绝时不能只有"不"** —— 拒绝 + 理由 + 折中 = 完整的拒绝
5. **对领导永远带结论上去** —— 汇报问题同时带上你的判断和建议
6. **每条回复都要过"精炼三问"** —— 能不能再短？第一句是否说清目的？有没有废话？（见文件顶部原则）
